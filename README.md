# Системный дизайн сервиса заметок

## Описание проекта
Проектирование масштабируемого сервиса для создания и управления заметками с поддержкой медиафайлов. Включает архитектурные решения, расчет нагрузки и выбор технологий хранения данных.

## Ключевые компоненты системы

### Архитектура
- **Фронтенд**: Пользовательский интерфейс с клиентской логикой
- **Бэкенд**: Обработка заметок, управление пользователями
- **Балансировщик нагрузки**: Распределение трафика между регионами
- **Кэш Redis**: Быстрый доступ к часто используемым данным
- **Очередь сообщений**: Сглаживание пиковой нагрузки

### Хранилища данных
- **PostgreSQL**: Метаданные пользователей и заметок
- **Amazon S3**: Медиафайлы (изображения, видео) и резервные копии
- **Redis**: Кэширование горячих данных

## Расчет нагрузки

### Пользовательская активность
- **DAU**: 20,000 пользователей
- **Пиковый RPS**: 10.8 запросов в секунду
- **Трафик**: 365.3 КБ/с
- **Хранилище**: 5 ТБ/год с учетом резервного копирования

### Характеристики системы
- **Read-heavy** нагрузка
- Приоритет доступности над строгой согласованностью (AP в CAP)
- Поддержка большого количества одновременных пользователей

## Особенности реализации
- Модульная архитектура для легкого расширения
- Разделение метаданных и медиафайлов
- Система резервного копирования
- Оптимизация для пиковых нагрузок
